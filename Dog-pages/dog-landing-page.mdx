# Dog Landing Page Module

The Dog Landing Page surfaces dog breed discovery, quick filters, size-based browsing, curated picks, comparisons, and relevant blog content. It is served at `/dog-breeds` and rendered with SSR.

## üß© Component Overview

- `Layout` and `Footer` for page chrome
- `Breadcrumbs` for navigation trail
- Hero banner with responsive images and a `SearchBar`:
  - Props: `petType="dog"`, `breed`, `setBreed`, `breedName`, `setBreedName`, `listFilter`, `setListFilter`, `breedListData`, `setBreedSlug`
  - Debounced search (500ms) triggers breed list fetch
- `MostPopularChips` as quick filters (Apartment friendly, Low shedding, Family friendly, Tiny/Large/Medium)
- Curations:
  - `IndianPremiumBreed` (Top premium breeds)
  - `Top4IndianBreeds` (Indian-origin breeds)
  - `SizeDoesMatter` (Small/Medium/Large/Giant)
  - Curated ‚ÄúTop breeds of the month‚Äù card grid
- Explore actions: compare breeds, breed selector, dog name generator (links/cards)
- Content: `ArticlesListCard` for dog blogs
- `FAQs` with schema

## üîÑ Data Flow

- SSR: `getServerSideProps` fetches:
  - `GET footer?footer_content=dog-landing-page` ‚Üí SEO metadata
  - `GET breed/common-data?page_name=dogLandingPage` ‚Üí counts, curated lists, dogStories
- CSR (on search/filter update):
  - Dispatches `breedList({ searchDebounce, filterBy, type: "dog", sortBy: "" })`
  - Endpoint: `GET breed/list?search=...&limit=500&filterBy=...&pet_type=dog&sortBy=...`
  - 500ms debounce for input

## üß≠ UX Flow

1. Page SSR renders hero with SEO-ready head tags (Organization, BreadcrumbList, ItemList, optional FAQ schema).
2. User can:
   - Use the search bar to find a breed; upon selection, navigate to `/dog-breeds/[slug]`.
   - Tap quick filters to pre-fill and refine the list.
   - Explore curated sections: Top breeds of the month, Indian climate breeds, Premium breeds.
   - Browse size-based sections via `SizeDoesMatter`.
   - Jump to compare tool, name generator, or breed selector.
   - Read featured Dog Blogs (first four items).
   - Consult FAQs.
3. Footer renders with global links and SEO content.

## üìà Performance & Behavior

- **SSR requests**: 2 (footer SEO + dog landing common data)
- **CSR requests**: Search/filter hits `breed/list` after 500ms debounce
- **Known heavy ops**:
  - `breed/list` with `limit=500`. If user typing causes frequent calls, rely on debounce; consider reducing limit or implementing pagination/‚Äúshow more‚Äù for lists.
- **Images**:
  - Served via `HAPPYPET_IMAGE_URL`; ensure proper dimensions and lazy loading where applicable.
- **Recommendations**:
  - Cache SSR `breed/common-data` briefly (CDN or server cache) to reduce TTFB variation.
  - Confirm hero/media assets are compressed and sized appropriately for mobile/desktop.
  - Monitor LCP on `/dog-breeds` for hero/banner images.
