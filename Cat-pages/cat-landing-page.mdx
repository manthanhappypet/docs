# Cat Landing Page Module

The Cat Landing Page helps users explore cat breeds via search, size filters, curated picks, comparisons, and blog content. It is served at `/cat-breeds` and rendered with SSR.

## üß© Component Overview

- `Layout`, `Footer` for page chrome
- `Breadcrumbs` for navigation trail
- Hero section with responsive banner and `SearchBar`:
  - Props: `petType="cat"`, `breed`, `setBreed`, `breedName`, `setBreedName`, `listFilter`, `setListFilter`, `breedListData`, `setBreedSlug`
  - 500ms debounced search triggers list fetch
- Curations:
  - `BasicBreedCard` list from `popularBreeds` (static types)
  - `IndianPremiumBreed` using `indianPremiumBreeds` (static types)
  - `SizeDoesMatter` using counts from SSR `landingPageData`
- Explore actions: compare breeds (grid and ‚ÄúCustom Comparison‚Äù link)
- Content: `ArticlesListCard` for Cat Blogs (from SSR `landingPageData.catStories`)
- `FAQs` with schema

## üîÑ Data Flow

- SSR (`getServerSideProps`) fetches:
  - `GET footer?footer_content=cat-landing-page` ‚Üí SEO metadata
  - `GET breed/common-data?page_name=catLandingPage` ‚Üí size counts, totalBreeds, catStories
  - Injects as `landingPageData = data.catLandingData`
- CSR (on search/filter update):
  - Dispatches `breedList({ searchDebounce, filterBy, type: "cat", sortBy: "" })`
  - 500ms debounce for input
- Unmount cleanup:
  - Dispatches `clearBreedListData()` to reset list state

## üì° API Endpoints

<AccordionGroup>
  <Accordion title="GET breed/common-data?page_name=catLandingPage">
    - **Purpose**: Provides landing data for cats (counts by size, totalBreeds, catStories)  
    - **Used In**: SSR to hydrate page content
    ```http
    GET /breed/common-data?page_name=catLandingPage
    ```
    - **Response (fields of interest)**:
      - `data.catLandingData.totalBreeds`
      - `data.catLandingData.breedCountsBySize.{totalSmallBreedsCount,totalMediumBreedsCount,totalLargeBreedsCount}`
      - `data.catLandingData.catStories[]`
  </Accordion>
  <Accordion title="GET breed/list">
    - **Purpose**: Search/filter cat breeds client-side  
    - **Triggered By**: Debounced search or quick filters  
    ```http
    GET /breed/list?search={q}&limit=500&filterBy={keyOrEmpty}&pet_type=cat&sortBy=
    ```
    - **Notes**: Large `limit` may affect payload; consider pagination if needed
  </Accordion>

## üß≠ UX Flow

1. SSR renders hero, search, and SEO-ready head tags (Organization, BreadcrumbList, ItemList, optional FAQ schema).
2. User can:
   - Search and select a breed ‚Üí navigates to `/cat-breeds/[slug]`.
   - Explore ‚ÄúMost popular Cat Breeds in India‚Äù (static curated list).
   - View premium breeds section.
   - Browse by size via `SizeDoesMatter` (Small/Medium/Large).
   - Use comparison grid or ‚ÄúCustom Comparison‚Äù.
   - Read Cat Blogs (first four items).
   - Read FAQs.
3. Footer renders with global links and SEO content.

## üìà Performance & Behavior

- **SSR requests**: 2 (footer SEO + cat landing common data)
- **CSR requests**: Search/filter hits `breed/list` after 500ms debounce
- **Known heavy ops**:
  - `breed/list` with `limit=500`; rely on debounce to minimize chatter
- **Images**:
  - Served via `HAPPYPET_IMAGE_URL`; ensure responsive sizing and lazy loading
- **Recommendations**:
  - CDN/server cache for `breed/common-data` to stabilize TTFB
  - Audit hero/banner assets for LCP on `/cat-breeds`

## üß™ SEO & Structured Data

## üîó Key Routes

- Landing: `/cat-breeds`

- Compare: `/cat-breeds/compare?breed1={slug}&breed2={slug}`
- All breeds list: `/cat-breeds/list`

## ‚ö†Ô∏è Edge Cases

- Empty search string resets to broad list; debounce avoids request spam.
- If `catStories` < 4, the blogs section is hidden.
- Size cards render only when corresponding counts > 0.
