# Bird Landing Page Module

The Bird Landing Page helps users explore pet bird species via search, curated picks, size browsing, comparisons, and FAQs. It is served at `/birds` and rendered with SSR.

## 📁 Location

- **Page**: `src/pages/birds/index.jsx`
- **Store (breeds)**: `src/store/breed.js`

## 🧩 Component Overview

- `Layout`, `Footer`, `Breadcrumbs`
- Hero with `SearchBar`:
  - Props: `petType="Bird"`, `breed`, `setBreed`, `breedName`, `setBreedName`, `listFilter`, `setListFilter`, `breedListData`, `setBreedSlug`
  - 500ms debounce triggers list fetch
- Curations:
  - “Popular Pet Birds in India” (static array within page)
  - “Top three Exotic Birds you can pet” via `IndianPremiumBreed`
  - “Popular Parrot Pets” via `Top4IndianBreeds`
  - “Browse by size” via `SizeDoesMatter` (Small/Medium/Large)
- Comparison grid + “Customize compare species”
- `FAQs` with schema

## 🔄 Data Flow

- SSR (`getServerSideProps`):
  - `GET footer?footer_content=bird-landing-page` → SEO metadata
  - `GET breed/common-data?page_name=birdLandingPage` → `landingPageData = data.birdLandingData` (size counts, totalBreeds)
- CSR (on search/filter):
  - Dispatch `breedList({ searchDebounce, filterBy, type: "bird", sortBy: "" })`
  - 500ms input debounce
- Navigation:
  - Selecting a species routes to `/birds/[slug]`

## 🌐 Environment Variables

- `HAPPYPET_API_URL` (API base)
- `HAPPYPET_IMAGE_URL` (assets base)
- `HAPPYPET_WEB_URL` (canonical/OG URLs)

## 🧭 UX Flow

1. SSR renders hero, search, curated sections, and SEO tags (schemas via `next/script`).
2. User searches or uses filters; list updates; selection navigates to `/birds/[slug]`.
3. Explore curated lists, size browsing, comparison, and FAQs.
4. Footer renders with global links/SEO.

## 📈 Performance & Behavior

- **SSR**: 2 calls (footer SEO + bird `common-data`)
- **CSR**: Debounced calls to `breed/list` (`limit=500`)
- **Images**: Served via `HAPPYPET_IMAGE_URL`; ensure responsive sizing and lazy loading
- **Recommendations**:
  - Cache `breed/common-data` on server/CDN
  - Audit banner and card images for LCP

## 🧪 SEO & Structured Data

- Schemas injected with `next/script`:
  - `Organization`, `BreadcrumbList`, `ItemList`, `FAQPage` (conditional)
- Meta: title, description, keywords, OG/Twitter, canonical to `/birds`

## 🔗 Key Routes

- Landing: `/birds`
- Species details: `/birds/[slug]`
- Compare: `/birds/compare?breed1={slug}&breed2={slug}`
- All species: `/birds/list`

## ⚠️ Edge Cases

- Blogs list is not present on birds landing (unlike dogs/cats).
- Size cards render only when counts > 0.
- Debounce prevents request spam on typing.

## 🧱 Imported Components (primary)
